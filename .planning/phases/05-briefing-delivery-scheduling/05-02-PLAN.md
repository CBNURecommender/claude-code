**File 2: `src/services/scheduler.py`**
Use python-telegram-bot's built-in JobQueue (per decision D-03) instead of separate APScheduler. This avoids event loop conflicts since JobQueue runs inside the bot's own event loop.

**Imports:**
- `json`, `datetime` from stdlib
- `from zoneinfo import ZoneInfo`
- `from telegram.ext import Application, ContextTypes`
- `from src.storage.database import get_setting, set_setting, get_db`
- `from src.delivery.telegram_sender import deliver_briefing, format_briefing_message`
- `from src.utils.logger import get_logger`

**Constants:**
- `KST = ZoneInfo("Asia/Seoul")`
- `logger = get_logger("services.scheduler")`

**Function: `async def job_briefing(context: ContextTypes.DEFAULT_TYPE) -> None`**
Scheduled briefing job callback. Called by JobQueue at configured times.
1. `bot = context.bot`
2. Try to import and call the summarizer: `from src.summarizer.briefing import generate_briefing`
3. Call `content, article_count, md_filepath = await generate_briefing()`
4. If `article_count == 0`: call `await deliver_briefing(bot, "", 0)` — deliver_briefing handles the "no articles" message
5. If `article_count > 0`:
   - Format timestamp: `datetime.datetime.now(KST).strftime("%Y-%m-%d %H:%M")`
   - Extract md_filename from md_filepath if not None (just the filename, not full path)
   - `msg = format_briefing_message(content, article_count, timestamp_str, md_filename)`
   - `await deliver_briefing(bot, msg, article_count)`
6. Wrap in try/except. On error, log `logger.error(f"Scheduled briefing failed: {e}")` and try to send error notification to all users: `await send_to_all_users(bot, f"⚠️ 브리핑 생성 중 오류가 발생했습니다: {e}")`

**Function: `async def job_collect(context: ContextTypes.DEFAULT_TYPE) -> None`**
Scheduled collection job callback. Called by JobQueue at configured intervals.
1. Try to import and call: `from src.collector import collect_all_sources` (Phase 2 provides this)
2. `count = await collect_all_sources()`
3. `logger.info(f"Scheduled collection completed: {count} new articles")`
4. On error: `logger.error(f"Scheduled collection failed: {e}")`

**Function: `async def update_briefing_schedule(application: Application) -> None`**
Read briefing_times from DB settings and update JobQueue jobs. Called on startup and after /set_times.
1. `job_queue = application.job_queue`
2. Remove existing briefing jobs: `jobs = job_queue.get_jobs_by_name("briefing"); for job in jobs: job.schedule_removal()`
3. Read times: `raw = await get_setting('briefing_times')` — parse as JSON array, default to `["08:00", "18:00"]` if None
4. For each time string in the array:
   - Parse hour, minute: `hour, minute = map(int, t.split(':'))`
   - `job_queue.run_daily(job_briefing, time=datetime.time(hour=hour, minute=minute, tzinfo=KST), name="briefing", data={"time": t})`
5. `logger.info(f"Briefing schedule updated: {times}")`

**Function: `async def setup_scheduled_jobs(application: Application) -> None`**
Called once from main.py after Application is built. Sets up both collection interval and briefing schedule.
1. `job_queue = application.job_queue`
2. Set up collection interval job:
   - `raw_interval = await get_setting('collection_interval_minutes')` — default `"30"` if None
   - `interval = int(raw_interval)`
   - `job_queue.run_repeating(job_collect, interval=interval * 60, first=10, name="collector")` (first=10 means start 10 seconds after boot, not immediately)
3. Set up briefing schedule: `await update_briefing_schedule(application)`
4. `logger.info("Scheduled jobs initialized")`

**Handler: `async def set_times_handler(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None`**
Handles `/set_times HH:MM [HH:MM] ...` command (DLV-02, DLV-03).
1. Parse args: `times = context.args` — if empty, reply with usage: `"사용법: /set_times 08:00 18:00"`
2. Validate each time string with regex `r'^\d{2}:\d{2}$'` and check `0 <= hour <= 23`, `0 <= minute <= 59`
3. If any invalid, reply with error: `"잘못된 시간 형식: {t}. HH:MM 형식으로 입력해주세요."`
4. Sort times, save to DB: `await set_setting('briefing_times', json.dumps(sorted(times)))`
5. Update schedule immediately (DLV-03): `await update_briefing_schedule(context.application)`
6. Reply with confirmation: `"✅ 브리핑 시간이 설정되었습니다: {', '.join(sorted(times))}"` followed by next briefing time info

**Handler: `async def briefing_handler(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None`**
Handles `/briefing` command for immediate briefing (DLV-04).
1. Reply: `"⏳ 브리핑을 생성하고 있습니다..."`
2. Call `await job_briefing(context)` — reuse the same scheduled job logic
3. The job_briefing function handles all the summarization, formatting, and delivery

**Handler: `async def collect_handler(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None`**
Handles `/collect` command for immediate collection.
1. Reply: `"⏳ 기사를 수집하고 있습니다..."`
2. Call `await job_collect(context)` — reuse the same scheduled job logic
3. After completion, reply with: `"✅ 수집 완료"` with article count if available
