**2. `/remove_source <name>` handler** (SRC-02):
Use ConversationHandler for two-step confirmation flow.

State constants at module level:
```python
CONFIRM_DELETE = 0
```

`async def remove_source_cmd(update, context)`:
- Check args has 1 argument. If not: reply "사용법: /remove_source <이름>"
- Look up source by name via `queries.get_source_by_name(name)`
- If not found: reply "소스를 찾을 수 없습니다: {name}"
- Get keyword count via `queries.list_source_keywords(source['id'])`
- Store `source` in `context.user_data['pending_delete']`
- Reply:
```
'{name}' 소스를 삭제하시겠습니까?
등록된 키워드 {len(keywords)}개도 함께 삭제됩니다.
삭제하려면 /confirm_delete 를 입력하세요.
취소하려면 /cancel 을 입력하세요.
```
- Return `CONFIRM_DELETE`

`async def confirm_delete_cmd(update, context)`:
- Get `source` from `context.user_data.get('pending_delete')`
- If None: reply "삭제할 소스가 없습니다." and return ConversationHandler.END
- Call `queries.remove_source(source['id'])`
- Clear `context.user_data['pending_delete']`
- Reply: "소스 삭제 완료: {source['name']}"
- Return `ConversationHandler.END`

`async def cancel_cmd(update, context)`:
- Clear `context.user_data.get('pending_delete', None)`
- Reply: "삭제가 취소되었습니다."
- Return `ConversationHandler.END`

**3. `/list_sources` handler** (SRC-03):
```python
async def list_sources_cmd(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None:
```
- Call `queries.list_sources()`
- If empty: reply "등록된 소스가 없습니다."
- Build formatted message following SOP Section 4-2 `/list_sources` format:
```
등록된 뉴스 소스 ({count}개)

1. {enabled_icon} {name}
   URL: {url}
   키워드: {keyword_list} ({keyword_count}개)
```
Where `enabled_icon` is "ON" if enabled else "OFF (비활성)", and keyword_list shows comma-joined keywords or "없음 (전체 수집)" if count is 0.
- For keyword display: call `queries.list_source_keywords(source['id'])` for each source

**4. `/enable_source <name>` handler** (SRC-04):
- Check args, look up by name
- If not found: Korean error
- If already enabled: "이미 활성 상태입니다: {name}"
- Call `queries.enable_source(source['id'])`
- Reply: "소스 활성화 완료: {name}"

**5. `/disable_source <name>` handler** (SRC-04):
- Same pattern, opposite direction
- Reply: "소스 비활성화 완료: {name}"

**6. `register_source_handlers(app: Application) -> None`:**
- Register ConversationHandler for remove_source flow:
```python
delete_conv = ConversationHandler(
    entry_points=[CommandHandler("remove_source", remove_source_cmd)],
    states={
        CONFIRM_DELETE: [
            CommandHandler("confirm_delete", confirm_delete_cmd),
            CommandHandler("cancel", cancel_cmd),
        ],
    },
    fallbacks=[CommandHandler("cancel", cancel_cmd)],
)
app.add_handler(delete_conv)
```
- Register simple CommandHandlers for add_source, list_sources, enable_source, disable_source
